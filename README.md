# 🎯 Deadline 冲刺器

> 大一升大二 · 面向对象程序设计 · 期末大作业
>
> 一个 C++ 规则怪谈游戏——**你的 OOP 大作业还有 4 天交，你能写完吗？**

---

## 🎮 这是什么

你是一个大一学生。课程群发消息——OOP 大作业下周五交。你认为下个月才交。

你翻身下床。室友老卷已经写完了。室友阿瓜在刷手机（他其实偷偷写完了）。室友老摸还在睡。

**你还有 4 天。**

- 去电脑桌写代码 → 推进度·涨焦虑
- 找老卷 B 问问题 → 推进度
- 找阿瓜 C 聊天 → 降焦虑
- 找老摸 D 取经 → 解锁 buff
- 喝咖啡 → 暂时提神（有空腹惩罚）
- 睡觉 → 推进下一天
- 去走廊 → 交作业（触发结局）

墙上贴着一张泛黄的纸——**6 条规则**。每条都可以违反，违反有惩罚。

## 🧱 代码架构

### 类继承图

```
MapObj（抽象基类）
  ├─ DeskObj     → onInteract() → doDesk()
  ├─ BedObj      → onInteract() → doBed()
  ├─ MateBObj    → onInteract() → talkToB()
  ├─ MateCObj    → onInteract() → talkToC()
  ├─ MateDObj    → onInteract() → talkToD()
  ├─ DoorObj     → onInteract() → doDoor()
  └─ CoffeeObj   → onInteract() → doCoffee()

Game（组合所有 MapObj*，协调全部逻辑）
AStarNode（寻路辅助）
```

### 设计模式

| 模式 | 位置 | 作用 |
|------|------|------|
| **多态** | `MapObj → 7 个派生类` | `o->onInteract(*this)` 一行替代 7 路 `strcmp` 分支 |
| **组合** | `Game` 持有 `MapObj*[]` | 数据 / 逻辑 / 规则 / 渲染四层在一个类里 |
| **开闭原则** | 加新物体 = 加新派生类 | Game 类不需要改动 |

## 📜 6 条规则

| # | 规则 | 触发条件 | 违反惩罚 |
|---|------|----------|----------|
| 0 | **不要空腹喝咖啡** | 没写代码就喝咖啡 | 胃痛，浪费整个时段 |
| 1 | **深夜不要重构** | 深夜对话选「重构旧代码」 | 进度 -5，焦虑 +10 |
| 2 | **不要连续写超过 2 次** | 第 3 次写代码 | 进度 -2，焦虑 +10 |
| 3 | **不要对老卷撒谎** | 被问进度时吹牛 | 老卷不再理你 |
| 4 | **Day 4 不能睡觉** | 交稿日选择睡觉 | 直接摆了结局 |
| 5 | **交了就有分** | — | （唯一不惩罚的规则） |

## 🏆 4 种结局

| 条件 | 结局 |
|------|------|
| 进度 ≥ 60 | 🏆 **通关** —「写得不错」 |
| 30 ≤ 进度 < 60 | 😅 **低空飘过** — 60 分 |
| 焦虑 ≥ 90 | 😰 **代码屎山** — 满屏注释 |
| 进度 < 30 或 Day 4 睡觉 | 🛏️ **摆了** — 下学期再见 |

## 💡 设计哲学

1. **约束激发设计**——只给 2 个数值（进度 + 焦虑），反而让每种行为的选择都意味深长
2. **规则不是弹窗**——玩家通过室友的反应、咖啡机的红灯、自己的手抖，*自己发现*规则
3. **多态用在刀刃上**——只在物体交互这一层用继承/多态，不为了「看起来 OOP」而强行加抽象
4. **先跑通再重构**——功能稳定后才引入多态（Replace Conditional with Polymorphism）
